Conclusion

Future Enhancements

We have tried to design a User Feedback System using the best possible combination of architectural styles available and using modern technologies. However, there are still some possible enhancements which can be done to further improve the design of the system:

  • We could potentially break the APIs into further smaller pieces, where each API would have single responsibility
  • We also could breakdown the datastore into multiple different databases which database used by one and only API. This promotes data governance and enhances the overall data model of the system
  • System is using the same database to perform transactional processing as well as analytical processing. Typically, this could result in poor end user experience when a resource heavy analytical processing is taking place. System could potentially be enhanced to have a data warehouse to support analytical processing
